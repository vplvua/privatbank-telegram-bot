## 1. Overview

### 1.1 Problem Statement

Власник ФОП рахунку в ПриватБанку не отримує миттєвих сповіщень про рух коштів у зручному форматі. Стандартні сповіщення від банку або відсутні, або незручні для щоденного моніторингу. Немає можливості швидко перевірити стан рахунку без входу в Приват24.

### 1.2 Solution

Автоматизований Telegram-бот, який periodically опитує API ПриватБанку та надсилає персональні сповіщення про кожне надходження або списання коштів з ФОП рахунку.

### 1.3 Goals

- Отримувати сповіщення про транзакції в Telegram в межах 5 хвилин після їх появи в API
- Не пропускати жодної транзакції
- Не надсилати дублікати
- Бот повинен працювати автономно без ручного втручання

### 1.4 Non-Goals (out of scope for v1)

- Аналітика витрат/надходжень
- Звіти за period
- Команди бота (відповідь на `/balance` тощо)
- Підтримка кількох рахунків
- Веб-інтерфейс

## 3. Functional Requirements

### 3.1 Transaction Polling

| ID | Вимога | Пріоритет |
|----|--------|-----------|
| FR-01 | Система опитує PrivatBank API кожні 5 хвилин | Must Have |
| FR-02 | При першому запуску зберігає поточний стан (без надсилання старих транзакцій) | Must Have |
| FR-03 | Порівнює нові транзакції з останньою збереженою позицією | Must Have |
| FR-04 | Надсилає повідомлення тільки для нових транзакцій | Must Have |
| FR-05 | Зберігає ID або timestamp останньої обробленої транзакції в persistent storage | Must Have |

### 3.2 Telegram Notifications

| ID | Вимога | Пріоритет |
|----|--------|-----------|
| FR-06 | Надходження коштів — окремий формат повідомлення (зелений індикатор 🟢) | Must Have |
| FR-07 | Списання коштів — окремий формат повідомлення (червоний індикатор 🔴) | Must Have |
| FR-08 | Повідомлення містить: суму, валюту, опис/призначення, дату та час | Must Have |
| FR-09 | Heartbeat повідомлення — раз на добу "бот живий" зі статусом останньої перевірки | Should Have |

### 3.3 Приклад формату повідомлення

```
🟢 Надходження
Сума: +5 000,00 UAH
Від: ТОВ "КЛІЄНТ"
Призначення: Оплата за послуги, рахунок №123
Час: 02.04.2026 14:32
```

```
🔴 Списання
Сума: -1 200,00 UAH
Отримувач: ФОП Іваненко І.І.
Призначення: Оплата за товар
Час: 02.04.2026 15:10
```

### 3.4 Error Handling

| ID | Вимога | Пріоритет |
|----|--------|-----------|
| FR-10 | Якщо PrivatBank API недоступний — логувати помилку, не крашити | Must Have |
| FR-11 | Якщо Telegram API недоступний — зберегти транзакції для повторного надсилання | Should Have |
| FR-12 | Якщо API токен протух — надіслати попередження в Telegram (якщо можливо) | Should Have |

## 5. Technical Constraints

- **PrivatBank API:** тільки polling, вебхуків немає
- **Auth:** Bearer токен з Приват24 Бізнес
- **Rate limits:** не надсилати запити частіше ніж дозволено API (уточнити в документації)
- **Telegram:** Bot API, sendMessage метод, chat_id захардкоджений або в env

## 6. Data Model

### Persistent State (мінімальний)

```json
{
  "lastProcessedAt": "2026-04-02T14:32:00Z",
  "lastTransactionId": "UA_PRIV_TXN_XXXXX"
}
```

Ключ унікальності транзакції: конкатенація `REF` + `REFN` (офіційна рекомендація ПриватБанку).

Зберігається в **Upstash Redis** (v1, через Vercel Integration) або JSON-файлі (після міграції на VPS).

> ℹ️ `@vercel/kv` задепрекейтований. Використовується `@upstash/redis` — офіційна заміна від Vercel.
